TicTacToe using Reinforcement Learning - Temporal Difference Method


Mageshwaran Anbazhagan


The project is about training an agent to play TicTacToe game successfully. The game is played by two players and each is assigned with a piece, either "X" or a "O". The agent is considered to the first player in this case, and performs the first move. The game terminates when the primary diagonal or secondary diagonal or entire row or entire column corresponds to a single player. If none of the instances or column converge to a winner, the game is drawn, with the basic overview of the game rules and procedure the agent is trained in the below part of the code using Temporal Difference Method (Reinforcement Learning).

The temporal difference method uses the value stored for the each action from the current state to train the agent make the best move. The value for the action is provided by using the following formula,

V(St) = V(St) + η[V(St+1) – V(St)]

where η, is the learning rate of the method. The value of the current state is determined by value from the future state. There is a reward provided to the model upon completion of the game. If the model wins the game a reward of "1" is provided to the state where the model declares itself as the winner and from there it backpropagates to update the state it took to reach the final state. Similarly, the model is given a "0" reward when it loses a game, but when there is no more move available and there is no clear winner, the game is considered to be drawn and the agent needs to be rewarded for the current state and thus a "0.6" is given. A slight advantage is given to the agent for arriving to the draw state, as the agent is trained to be more defencive and avoid losing the game.

Evaluating the presence of a winner


Once the game is started, the agent and the player 2 start battling against each other to showcase their upper hand on each other, to determine who have crossed the line before the other, the checkWinner() function is used. Inside this function, a gameOver flag with value "0" is initialized to indicate the presence of a winner. The list of values in the board environment is converted to a 3X3 array with the help of numpy package and calculated for the diagonal, row and column level values. If any of the three consecutive cell have the same player or agent values, they are considered to be the winner. If the environment predicts a winner, the gameOver flag is set to "1" and the value is then returned to the calling function.

def checkWinner():
    gameOver = 0
    boardMatrix = np.array(board)
    shape = (3,3)
    boardMatrix = boardMatrix.reshape(shape)
    primaryDiagonal = np.trace(boardMatrix) ** 2
    secondaryDiagonal = np.sum(np.fliplr(boardMatrix).diagonal()) ** 2
    firstRow = np.sum(boardMatrix[0]) ** 2
    secondRow = np.sum(boardMatrix[1]) ** 2
    thirdRow = np.sum(boardMatrix[2]) ** 2
    firstColumn = np.sum(boardMatrix[:,0]) ** 2
    secondColumn = np.sum(boardMatrix[:,1]) ** 2
    thirdColumn = np.sum(boardMatrix[:,2]) ** 2
    if 9 in {primaryDiagonal, secondaryDiagonal, firstRow, secondRow, thirdRow, firstColumn, secondColumn, thirdRow}:
        gameOver = 1
    return gameOver


Enter the game to train the agent


The startGame() function initiates the game by asking the agent to plot his first move, since the one assigned with "X" always starts the game. The agent starts his learning from the basic with zero or no knowledge on playing the game. A boardState variable is created where for every the state the reward calculated is updated on completion of the game. Every time the agent completes the game, the steps taken are recorded and backpropagated to update the values for being in that state using the reward earned at the end of each game. When the game gets started the posAvailable() function is invoked to get the agent know about the available playing environment. Once the positions are known, the agent checks the boardState variable to determine his next step. The highest value in the data set which is also available is considered to be the best step for the agent. Once the agent has completed his turn, the turnOverP1 variable is set to 1 indicating the usage of his turn. A randomint function is used to predict the next move of the player 2 upon validating with the available position list. One every move of the players, the checkWinner() function is invoked such that a winner is not asked to continue the game further. If the agent is classified as the winner, a reward of 1 is given, a reward of 0 is given, if he loses, and a reward of 0.6 is given, if he draws the game. The reward value is used as input to the stateVal() function invoked here.

Assisting the agent to decide his future moves


The stateVal() function takes in the reward value from the startGame() function and updates the boardState data set of the agent, such that the agent learns from his mistakes. The function also gets the states of the agent upon completion of the game. Using the playerState list, the function backpropagates and updates the value in the boardState data set with the formula provided in the top. The η in the formula is taken to be 0.4, which is taken randomnly.

def stateVal(reward):
    flag = 0
    eeta = 0.4
    for i in reversed(playerState):
        if flag == 0:
            boardState[i-1] = reward
            flag = 1
            update = i
        else:
            boardState[i-1] = boardState[i-1] + eeta * (boardState[update - 1] - boardState[i-1])
            update = i
